# Contributing

This project is a curated working library, not a complete directory.

## What Belongs Here

Add projects that help developers use AI to build software more effectively. Good entries usually support one of these workflows:

- AI coding agents and IDE assistants
- Codebase search, indexing, and context retrieval
- Model Context Protocol tools and servers
- Browser, terminal, shell, and UI automation for agents
- Testing, evals, and verification of AI-generated changes
- Local model serving and model-routing infrastructure
- Sandboxes and secure execution for agent-written code
- Documentation, RAG, and knowledge tools used by engineering teams

## What Does Not Belong Here

- Generic open source libraries with no AI development workflow angle
- Closed-source products with only a thin SDK or demo repo
- Prompt dumps without maintained code
- Old "awesome list" style categories such as CMS, database, frontend framework, or CLI tool unless the project directly improves AI-assisted development
- Projects added only because they are popular

## Entry Format

Use this format:

```markdown
- [Project](https://github.com/org/project) - Short, concrete description. `Language` `License`
```

Rules:

- One sentence only.
- Say what the tool does in a developer workflow.
- Do not include star counts.
- Do not use hype words such as "revolutionary", "best", or "ultimate".
- Link to the canonical repository or organization page.

## Review Checklist

Before adding an entry, check:

- The project is open source or has a meaningful open source core.
- The repository is still usable.
- The description is specific enough to distinguish it from nearby tools.
- The category is the most useful place for a maintainer to find it.
- The project helps AI-assisted development, not just software development in general.

## Category Changes

When adding a new category, include a short reason in the pull request. Prefer expanding existing categories until there are enough entries to justify a split.
